BrightSDK / React Native Plugin

Bridge BrightSDK into React Native without rebuilding the native layer in every app.

react-native-bright-sdk exposes BrightSDK initialization, consent management, and reporting through a compact JavaScript API backed by native Android and Windows modules.

Why this plugin exists

It removes repetitive platform bridge work from app teams. The JavaScript surface stays stable while the Android and Windows implementations absorb BrightSDK native integration details.

Native bridge included

No app-specific module plumbing for Android or React Native Windows.

  • Android package registration is provided.
  • Windows ships a C++/WinRT module.
  • Autolinking metadata is already part of the package.

Consent flow control

App code decides when consent is shown and how the choice should be applied.

  • enable() maps to opt-in.
  • disable() maps to opt-out.
  • reportConsentShown() reports the UI event.

SDK packaging handled

The Android library is already wired to the BrightSDK Gradle plugin.

  • SDK artifacts are installed before build.
  • Consumer apps only add the npm package.
  • Native SDK changes stay centralized here.

Consistent app API

Application code uses one JavaScript module regardless of target platform.

  • Shared startup call.
  • Shared consent enable/disable model.
  • Future iOS and macOS support can preserve the same API.

Install and initialize

Use a release tarball locally or install directly from the repository when validating changes.

Install

npm install ./react-native-bright-sdk-2.0.3.tgz

# or

npm install git+https://github.com/BrightSDK/react-native-plugin.git

Initialize in app code

import { useEffect } from 'react';
import BrightSdk from 'react-native-bright-sdk';

export default function App() {
  useEffect(() => {
    BrightSdk.init();
  }, []);

  return null;
}
Android setup note Ensure your root Gradle build can resolve com.brightdata:bright-sdk-gradle:1.+ and that your manifest includes internet-related permissions. The plugin handles the BrightSDK AAR installation step.
Windows setup note Run npx react-native autolink-windows after installation and place lum_sdk.dll next to your app executable so the native module can load it.

JavaScript API

The public API stays intentionally small so app code can integrate BrightSDK with minimal conditional logic.

init()

Initializes BrightSDK using platform defaults. Call once near app startup.

enable()

Applies an opt-in decision through the native platform implementation.

disable()

Applies an opt-out decision and shuts down peer activity as required.

reportConsentShown()

Notifies the SDK that the consent prompt was shown to the user.

getConsentChoice()

Returns the current consent state as true, false, or null.

getUuid()

Retrieves the SDK-generated UUID when available.

close()

Closes the SDK and releases resources. On Windows this also unloads the DLL.

setAppId()

Windows-only helper for setting the application ID before initialization.

Architecture snapshot

Repository structure keeps the JS contract thin and pushes platform-specific work into native modules.

Android path

BrightSdkNativeModule.java

Exposes JS-callable methods with React Native annotations.

BrightSdkHelper.java

Wraps the BrightSDK Android APIs and initialization defaults.

android/build.gradle

Applies the BrightSDK Gradle plugin and ensures SDK installation before build.

Windows path

BrightSdkModule.vcxproj

Defines the React Native Windows native module project.

C++/WinRT bridge

Delegates enable, disable, consent, and UUID calls into the BrightSDK DLL entry points.

Autolinking metadata

Package metadata registers the Windows project with React Native tooling.
